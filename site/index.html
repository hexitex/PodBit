How It Works

Three sources of knowledge. One engine.

Seed from three sources

Your documents are one input - drop folders of PDFs, spreadsheets, code, and images. But the LLMs already carry the knowledge of the internet in their weights, and synthesis draws on that too. Then the research agent identifies gaps, searches the web for evidence, and seeds what it finds back into the graph. Your material, LLM knowledge, and live research all feed the same engine.

Evolve through synthesis

The genetic algorithm samples node pairs by embedding similarity, voices novel connections through LLMs, and creates child nodes that inherit from both parents. The LLM doesn't just rearrange your words - it brings its own training knowledge into every synthesis. Multi-parent recombination. Temperature curves that cool over generations. Niching pressure that prevents domain collapse. Knowledge that survives is stronger than anything a single prompt produces.

Verify with executable proof

Every synthesis output passes through nine quality gates - injection detection, structural validation, novelty filtering, hallucination heuristics, claim provenance, deduplication, junk filtering, specificity enforcement, and domain concentration checks. What survives faces the EVM: it writes Python code to computationally test claims, runs it in a sandbox, and adjusts node weight based on the result.

This is not confidence scoring - it's proof.

Numbers don't cross domains.

LLMs leak numbers across domains during synthesis — a ratio from biology becomes a "constant" in physics. Podbit strips numbers at ingestion, stores them in a scoped registry, and forces synthesis to work blind. Cross-domain connections are structural or they don't exist. Verification gets the real numbers back to check the maths.

Document breakthroughs automatically

When a node earns breakthrough status, Podbit curates everything behind it - the parent nodes, the synthesis chain, the verification results, the domain context. It assembles this into a structured brief and hands it to a frontier model to produce fully audited documentation. You get a complete, traceable record of how the insight was discovered, what evidence supports it, and why it matters.

Serve to every model in your stack

Evolved knowledge flows through three channels: an industry-standard chat completions proxy that enriches requests automatically, an MCP server with 31 tools for direct graph access, and a REST API for custom integrations. The context engine tracks conversation topics across turns, detects intent, and selects the most relevant graph knowledge for each message. Cross-session learning means your second conversation is smarter than your first.

The System

Autonomous. Verifiable. Tunable.

Seven cycles running continuously. Twenty-nine agents, each with its own model, prompt, and tuning profile. Every aspect observable and adjustable.

Autonomous Cycles Synthesis, validation, tensions, questions, research, autorating, and EVM verification - all running without intervention.
AI Agents Each subsystem is a separate agent with its own model assignment, prompt, temperature profile, and quality thresholds.
Quality Gates Injection detection, structural validation, novelty, hallucination, provenance, dedup, junk, specificity, and concentration.
Tunable Parameters Mutation rates, gating thresholds, temperature curves, scoring weights, fitness functions. Organised across 45 sections with audit trails.
File Readers PDF, DOCX, XLSX, images (vision), code, and text. Content-hash change detection. Real-time folder watching.
MCP Tools Full graph access from any MCP-compatible client. Query, propose, voice, verify, compress, tune - everything exposed.

29 subsystems. Three tiers. One pipeline.

Every task is a separate AI agent. Assign frontier models where reasoning depth matters. Assign open-source weights everywhere else - most subsystems run brilliantly on a 20B at zero cost.

Frontier
Claude, GPT-4.5+, DeepSeek

Deep reasoning, novel synthesis, code generation. 11 subsystems including voicing, chat, research, docs, breakthrough validation, EVM codegen and analysis.

Medium
Qwen 32B, Llama 70B, GPT-4o-mini

Structured extraction, classification, curation. 11 subsystems including synthesis, compress, config tuning, EVM triage, and all 6 file readers.

Small
Llama 8B, Qwen 7B, Gemma 9B, Phi-3

Keywords, dedup, autorating, context compression, proxy routing. 5 subsystems. Often runs locally at zero cost.

Dedicated
nomic-embed, mxbai-embed

Embedding and image generation. 2 specialised non-chat models.

Control costs per subsystem

Every one of the 29 agents has its own model assignment. Route frontier models to the subsystems that need deep reasoning - voicing, research, breakthrough validation. Run everything else on local open-source weights at zero cost. Reassign any subsystem to a different model in one click, and see the impact immediately.

Most Podbit deployments run 20+ subsystems on free local models. You pay for frontier calls only where the quality difference justifies it.

Self-tuning via gold standards

Frontier models define what excellent output looks like for each subsystem. Smaller models are scored against these benchmarks. Temperature, model parameters, and quality thresholds auto-adjust - reducing frontier dependency while maintaining quality.

When a subsystem agent's confidence drops below threshold, it doesn't guess - it escalates. A more capable model reviews the uncertain output, provides guidance, and returns control. Escalation is automatic, per-subsystem, and logged.

Podbit is AI.

This isn't a tool that uses AI - it is AI. The genetic algorithm at its core breeds knowledge through recombination, mutation, fitness selection, and niching - producing emergent insight no single prompt or model would generate. Twenty-nine specialised agents, each with consulting escalation, each independently tuned, each validated against frontier-grade gold standards.

Chat, Control, Index

Your knowledge, accessible everywhere

The built-in GUI chat has full access to all 29 MCP tools. Ask questions, steer the engine, and manage your graph from the chat.

Chat with your knowledge

Ask questions and get answers grounded in your graph - not generic training data. Query nodes, compress domain knowledge into context, check graph health, inspect tensions and contradictions, and surface research gaps. Every response is traceable back to specific nodes with provenance chains.

Steer the engine

Force new synthesis combinations. Create and bridge domains. Promote breakthroughs. Tune the 298 algorithm parameters. Start and stop autonomous cycles. Full control over the genetic algorithm - seed ideas, verify claims, and shape the direction of evolution in real-time.

Enrich your AI augmented IDE through MCP

If you work in an MCP-compatible IDE, the same 31 tools are available there. Your coding agent gets direct access to domain knowledge, synthesis history, and verified breakthroughs - same capabilities, your choice of interface.

Index your codebase

Point the Knowledge Base at your source folders. The dual-folder pattern indexes your code twice: once as raw content for precise retrieval, and once as AI-curated knowledge - where an LLM reads your code, explains what it does, and creates searchable, synthesisable descriptions. Your codebase becomes part of the graph, discoverable alongside your research.

The orchestrator keeps it all alive

A background orchestrator monitors the API server, MCP connections, and all autonomous cycles. If a service drops, it restarts. If a connection breaks, it reconnects. You don't babysit infrastructure - you seed knowledge and let the system run.

Collaboration & Trust

Projects, partitions, and cryptographic trust

Knowledge is organised into projects with isolated domains. Guest knowledge enters through transient partitions - seeding new insight only when you approve it. Every node is hash-chained.

Project Isolation

Each project gets its own database, domains, manifest, and synthesis cycles. Switch between research contexts without cross-contamination. Each project's knowledge stays isolated until you explicitly bridge domains.

Transient Partitions

Import guest knowledge from a central repository or another Podbit instance. Transient partitions bridge on arrival and depart when done - the guest data seeds new synthesis only if you approve it. Nothing enters your graph without your consent.

Merkle Chain Integrity

Every node is SHA-256 hashed - content, type, contributor, timestamp, and parent hashes combined into a cryptographic content hash. A tamper-evident integrity log chains every operation. Exports carry a Merkle root; imports verify it. If a hash doesn't match, you'll know.

Trust without trusting.

Guest knowledge is useful. Unverified guest knowledge is dangerous. Podbit's integrity system lets you import external partitions, run synthesis across them, and verify that nothing was silently modified. The hash chain is the proof.

What Happens

The engine doesn't sleep.

You seed two domains. Bridge them. Walk away. Here's what happens next.

T+0

You seed 40 nodes across two domains and bridge them.

T+2 min

Synthesis engine pairs nodes by embedding similarity. First child nodes appear.

T+8 min

Tension cycle finds contradictions between your seeds and what the LLM knows. Question cycle generates research gaps.

T+15 min

EVM writes Python to verify a cross-domain claim. The math checks out. Node promoted.

T+30 min

Research agent searches the web for evidence supporting 3 research gaps. It generates its own nodes from what it finds - knowledge you never provided.

T+2 hr

Breakthrough detected. Three-gate pipeline confirms novelty. Registered in shared breakthrough registry.

T+overnight

You return to many more generations, and high-quality breakthroughs if found - knowledge no prompt would have produced.
